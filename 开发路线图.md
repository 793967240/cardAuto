>
> 目标：把 GDD §12 的 5 阶段 MVP 路径**落地为可执行的工程任务清单**，每条任务可直接转成 GitLab Issue。
>
> **EA 节点**：阶段 3.5 完成时上架 Steam EA。

---

## 关键决策（v0.2 起）

- **首发平台**：Steam（Windows / Linux / macOS），**PC 从阶段 1 起就是主开发目标**
- **安卓**：仅 APK 自测包，不上架商店；每月构建一次能跑即可
- **商业化**：纯单机买断，无账号 / 无内购 / 无云服务后端，仅依赖 Steam Cloud
- **EA 内容范围**：阶段 3 完成 + Act 1+2（**至少 2 个 Act**），1-2 个角色

---

## 总览：里程碑节奏

| 阶段 | 别名 | 工程目标 | 体感目标 | 预计周期 |
|------|------|---------|---------|---------|
| 0 | **地基** | 项目骨架 + CI + 测试框架 + 美术 pipeline + i18n 通水 | 无可玩内容 | 2-3 周 |
| 1 | **核心循环 MVP（PC）** | 一场可玩战斗（含构筑），PC 键鼠为基准 | "时间轴卡牌+自动战斗"爽不爽？ | 4-6 周 |
| 2 | **构筑深度** | 宝石循环 + 底座构筑 + 扩卡池 | "构筑思考深度够不够？" | 3-4 周 |
| 3 | **Roguelike 外壳** | 分叉爬塔 + 商店 + 事件 + 遗物 | "一个完整 Run 通顺不通顺？" | 4-5 周 |
| **3.5** | **EA 准备期** | Steam 集成 / 商店页 / 调优 / 体验闭环 | 体面地公开发售 | **3-4 周** |
| **🚀 EA** | **Steam Early Access 发布** | 收数据 / 修 bug / 调平衡 | 玩家口碑回流 | 持续 |
| 4 | **多角色 + Meta**（EA 期间持续更新） | 3 个新角色 + 元进度 + 平衡打磨 | "二刷意愿强不强？" | 6-8 周 |
| 5 | **1.0 正式发布** | 性能优化 + 完整内容 + 移动端评估 | 上架移动商店 / 1.0 庆祝 | 4-6 周 |

**重要**：每个阶段结束有**可玩 demo + 平衡报告 + 风险评估**，决定是否进入下阶段（或回头打磨）。

## 阶段 0：地基（2-3 周）
