# 本地化架构（i18n v0.1）

> 配套文档：[游戏设计文档.md](./游戏设计文档.md) · [技术设计文档.md](./技术设计文档.md) · [开发路线图.md](./开发路线图.md)
>
> 工具栈：**Godot 内置 `tr()` + CSV + Google Sheet 协作**
>
> EA 阶段语言：**简体中文 + 英文**
>
> 1.0 候选语言：**繁体中文 / 日文 / 韩文 / 俄文**（视社区呼声）

---

## 一、为什么阶段 0 必须落地

### 1.1 EA 上 Steam = 必须英文

Steam 全球玩家中：
- 中文玩家约 30%
- 英文玩家约 50%
- 其他语言约 20%

**没有英文 = 直接砍掉一半潜在用户**。EA 阶段双语是最低线。

### 1.2 不在阶段 0 落地的代价

| 时机 | 代价 |
|------|------|
| 阶段 0 | 增加 3-5 天工作量；**所有后续 UI 自然走 i18n** |
| 阶段 3 EA 前补 | 至少 2-3 周返工，要审查每个 UI 文件的硬编码 |
| EA 上线后补 | 玩家差评 + 翻译质量参差 + 兼容旧存档复杂 |

**结论**：阶段 0 必须强制落地，CI 加守卫规则。

### 1.3 i18n ≠ 仅翻译

本地化包含但不限于：
- 文本翻译（UI / 卡牌名 / 卡牌描述 / 事件 / 提示）
- **字体**（中文需子集化思源宋体，英文用 Cinzel / Cormorant）
- **数字 / 日期格式**（千分位 1,000 vs 1.000）
- **阅读方向**（中英都是 LTR，无影响；为日韩留接口）
- **图片中的文字**（卡牌图本身**禁止内嵌文字**，文字全走 UI 层）

## 二、Godot 4.x i18n 机制速览

### 2.1 内置 `tr()` 函数

```gdscript
# 任何字符串显示前必须经过 tr()
label.text = tr("battle.button.surrender")
# 实际显示：根据当前语言 → "投降" / "Surrender"
```

### 2.2 自动加载翻译资源

`project.godot`：

```ini
[internationalization]
locale/translations=PackedStringArray(
    "res://translations/zh_CN.translation",
    "res://translations/en.translation"
)
locale/fallback="en"
```

### 2.3 运行时切换语言

```gdscript
# src/globals/settings.gd
func set_language(locale: String) -> void:
    TranslationServer.set_locale(locale)
    EventBus.language_changed.emit(locale)
```

UI 层订阅 `language_changed` 信号刷新文本。

## 三、CSV 工作流

### 3.1 为什么选 CSV

- Godot 内置支持，编辑器可直接 import 为 `.translation`
- 文本编辑器 / Excel / Google Sheet 都能编辑
- Git diff 友好（行级变更可读）
- 不依赖任何第三方 SaaS

### 3.2 文件结构

```
translations/
├── _source.csv              # 主源文件（中文为基准源语言）
├── zh_CN.csv                # Godot 自动从 _source 派生（或人工维护）
├── en.csv                   # 英文翻译
├── zh_CN.translation        # Godot 编辑器自动 import 出来的资源
└── en.translation
```

### 3.3 CSV 格式

```csv
keys,zh_CN,en
battle.button.surrender,"投降","Surrender"
battle.button.pause,"暂停","Pause"
battle.button.resume,"继续","Resume"
battle.label.recovery,"调息","Recovery"
card.zhan.name,"斩","Sword Strike"
card.zhan.desc,"造成 {damage} 点伤害","Deal {damage} damage"
card.zhan_plus.name,"斩+","Sword Strike+"
card.zhan_plus.desc,"造成 {damage} 点伤害","Deal {damage} damage"
event.altar.title,"祭坛","The Altar"
event.altar.body,"你来到一个古老的祭坛。它要求献祭，作为交换给予力量。","You arrive at an ancient altar. It demands sacrifice in exchange for power."
```

### 3.4 占位符规则

```
{damage}        ← 数值占位（int / float）
{count}         ← 数量占位（影响复数处理）
{card_name}     ← 卡牌名占位
{element}       ← 元素属性占位（火/水/木...）
```

**规则**：
- 占位符必须在所有语言版本中**完全一致**（CI 校验）
- 不允许语言间增减占位符
- 占位符顺序可变（不同语言语序不同）

## 四、Key 命名规范

### 4.1 命名结构

```
{scope}.{component}.{purpose}.{detail}
```

### 4.2 完整示例

| Scope | 用途 | 示例 |
|-------|------|------|
| `battle` | 战斗界面 | `battle.button.surrender`, `battle.label.tick` |
| `build` | 构筑界面 | `build.tooltip.recovery`, `build.synergy.fire_resonance` |
| `map` | 爬塔地图 | `map.node.battle`, `map.node.shop` |
| `shop` | 商店 | `shop.title`, `shop.button.buy`, `shop.confirm.purchase` |
| `event` | 事件 | `event.altar.title`, `event.altar.option_a` |
| `card` | 卡牌 | `card.zhan.name`, `card.zhan.desc` |
| `enemy` | 敌人 | `enemy.yao_qi.name`, `enemy.yao_qi.flavor` |
| `relic` | 遗物 | `relic.ming_yu_pei.name`, `relic.ming_yu_pei.desc` |
| `trait` | 词条 | `trait.cost_minus_1.name` |
| `status` | 状态 | `status.weakness.name`, `status.weakness.desc` |
| `ui` | 通用 UI | `ui.button.confirm`, `ui.button.cancel` |
| `menu` | 菜单 | `menu.main.start`, `menu.settings.language` |
| `tutorial` | 教学 | `tutorial.step_01`, `tutorial.step_02` |
| `error` | 错误 | `error.save_corrupted`, `error.network` |
| `meta` | 元数据 | `meta.character.sword.name`, `meta.character.sword.bio` |

### 4.3 命名规则（强制）

- **全小写**，单词间用 `_`
- **scope 在最前**，便于分组
- **不要硬编码数字**（用 `step_01` 不要 `1`）
- **不要在 key 中放翻译**（不要 `button.tou_xiang_zhe`）
- **复数变体用后缀**：`item_count.one`, `item_count.other`

### 4.4 与卡牌 ID 的对齐

卡牌资源 ID（如 `sword_zhan`）与 i18n key 的关系：

```
ID:          sword_zhan
卡牌名 key:   card.sword_zhan.name
卡牌描述 key: card.sword_zhan.desc
```

→ 由 ID 自动派生 key，避免手动维护两份。

## 五、Google Sheet 协作工作流

### 5.1 为什么用 Sheet

- 多人同时编辑（翻译者 + 校审 + 程序员）
- 评论 / 历史记录 / 颜色标记齐全
- 可导出 CSV，无 vendor lock-in
- 主播 / 玩家也可帮忙翻译（开公开评论）
- AI 翻译辅助（Sheet 公式直接调 GPT API）

### 5.2 Sheet 结构

| key | zh_CN | en | en_status | en_translator | notes |
|-----|-------|-----|-----------|---------------|-------|
| battle.button.surrender | 投降 | Surrender | reviewed | @alice | - |
| card.zhan.name | 斩 | Sword Strike | reviewed | @alice | - |
| card.zhan.desc | 造成 {damage} 点伤害 | Deal {damage} damage | draft | AI | needs review |

**status 字段值**：
- `empty` - 还没翻译
- `draft` - AI 起草 / 翻译者初稿
- `reviewed` - 校审通过
- `final` - 上线就绪

### 5.3 同步流程

```
策划在主源 sheet 增删改 zh_CN
    ↓
翻译者填 en（draft 状态）
    ↓
校审改成 reviewed
    ↓
导出 CSV → Git commit translations/_source.csv
    ↓
CI 触发：CSV → Godot .translation 资源
    ↓
游戏内立即生效
```

### 5.4 AI 翻译辅助（Sheet 公式）

```
=GPT_TRANSLATE(B2, "en", "context: card name in a Chinese cultivation card battler game")
```

通过 Google Apps Script 接 GPT-4 / Claude / Gemini API，提供风格一致性提示。

**注意**：AI 翻译**必须人工校审**才能改为 `reviewed`。修仙术语 AI 经常翻译失准。

### 5.5 修仙术语词表（Style Guide）

附在 Sheet 第一个 tab "Glossary"：

| 中文 | 英文（统一） | 备选（拒绝） |
|------|------------|-----------|
| 修仙 | Cultivation | Immortal cultivation, Xianxia |
| 修整 | Recovery | Rest, Cooldown |
| 链条 | Chain | Sequence, Combo |
| 时间轴 | Timeline | Time track |
| 卷轴 | Scroll | Roll |
| 灵气 | Qi / Spiritual energy | Mana |
| 道伤 | Dao Wound | Spiritual injury |
| 业火 | Karmic Fire | Burning |
| 调息 | Adjusting Breath | Meditation, Resting |
| 剑意 | Sword Intent | Sword spirit, Sword aura |
| 丹气 | Pill Essence | Alchemy energy |
| 符箓 | Talisman | Charm, Spell paper |
| 渡劫 | Tribulation | Trial |

→ 翻译者必须查表，禁止自由发挥。

## 六、字体方案

### 6.1 中文字体

**首选：思源宋体（Source Han Serif）**
- 开源 OFL 商用免费
- 衬线风格匹配水墨修仙
- 多字重（Light / Regular / Bold）

**子集化处理**：
- 全字符集：~70 MB（包体灾难）
- 中文常用 3500 字 + 卡牌专用术语：~5 MB
- 工具：`pyftsubset`（Python）或 Glyphhanger

```bash
# tools/font_subset.py
pyftsubset SourceHanSerifSC-Regular.otf \
    --text-file=tools/font_chars.txt \
    --output-file=fonts/serif_zh.subset.otf \
    --flavor=woff2
```

`tools/font_chars.txt` 包含：
- GB2312 常用 3500 字
- 卡牌名所有汉字
- 事件文本所有汉字
- 数字 + 标点

### 6.2 英文字体

**首选：Cormorant Garamond**（衬线，古典感）
**备选**：Cinzel, EB Garamond
- Google Fonts 开源 OFL
- 全字符集 ~200 KB

### 6.3 字体加载策略

```gdscript
# src/globals/font_loader.gd
const FONTS = {
    "zh_CN": "res://assets/fonts/serif_zh.subset.otf",
    "en":    "res://assets/fonts/cormorant_garamond.ttf",
}

func get_font_for_locale(locale: String) -> Font:
    return load(FONTS.get(locale, FONTS["en"]))
```

UI 节点动态绑定 Theme 字体，语言切换时同步切换。

### 6.4 字体设计基线

| 场景 | 字号（1080p） | 字重 |
|------|------------|------|
| 卡牌名 | 24 px | Bold |
| 卡牌描述 | 18 px | Regular |
| UI 按钮 | 22 px | Regular |
| 事件正文 | 20 px | Regular |
| 标题 | 36 px | Bold |
| 数值（HP / 伤害） | 28 px | Bold |

→ 4K 显示器下按比例放大；小屏（Steam Deck）按 0.85x 缩放。

## 七、CI 校验

### 7.1 缺失 key 检测

```python
# tools/i18n_check.py
import csv
import re
import sys
from pathlib import Path

def find_all_tr_keys(src_dir):
    """扫描所有 .gd 文件，提取 tr("...") 中的 key"""
    keys = set()
    pattern = re.compile(r'tr\(\s*["\']([\w\.]+)["\']\s*\)')
    for f in Path(src_dir).rglob("*.gd"):
        keys.update(pattern.findall(f.read_text()))
    return keys

def find_all_csv_keys(csv_path):
    with open(csv_path) as f:
        return {row[0] for row in csv.reader(f) if row[0] != "keys"}

if __name__ == "__main__":
    code_keys = find_all_tr_keys("src")
    csv_keys = find_all_csv_keys("translations/_source.csv")

    missing = code_keys - csv_keys
    if missing:
        print(f"❌ Missing keys in translation CSV:")
        for k in sorted(missing): print(f"  - {k}")
        sys.exit(1)

    unused = csv_keys - code_keys
    if unused:
        print(f"⚠️  Unused keys in CSV (warning only):")
        for k in sorted(unused): print(f"  - {k}")
```

### 7.2 占位符一致性检测

```python
def check_placeholders(csv_path):
    """检查所有语言版本的占位符是否一致"""
    pattern = re.compile(r'\{(\w+)\}')
    with open(csv_path) as f:
        rows = list(csv.reader(f))
    headers, *data = rows
    for row in data:
        key = row[0]
        placeholders_per_lang = [
            (headers[i], set(pattern.findall(row[i])))
            for i in range(1, len(row)) if row[i]
        ]
        if not placeholders_per_lang: continue
        baseline = placeholders_per_lang[0][1]
        for lang, ph in placeholders_per_lang[1:]:
            if ph != baseline:
                print(f"❌ Placeholder mismatch in {key}:")
                print(f"   {placeholders_per_lang[0][0]}: {baseline}")
                print(f"   {lang}: {ph}")
                sys.exit(1)
```

### 7.3 硬编码字符串守卫（最严）

```python
# tools/i18n_no_hardcode.py
# 扫描所有 UI 脚本中是否有 .text = "中文/英文" 这类硬编码

import re, sys
from pathlib import Path

# 允许的例外（debug 文本、占位符等）
ALLOWLIST = {"src/debug/", "src/tools/"}

# 扫描 .text = "..." / .tooltip_text = "..." 等赋值
patterns = [
    re.compile(r'\.text\s*=\s*"([^"]+)"'),
    re.compile(r'\.tooltip_text\s*=\s*"([^"]+)"'),
    re.compile(r'\.placeholder_text\s*=\s*"([^"]+)"'),
]

def scan(src_dir):
    violations = []
    for f in Path(src_dir).rglob("*.gd"):
        if any(skip in str(f) for skip in ALLOWLIST): continue
        text = f.read_text()
        for p in patterns:
            for m in p.finditer(text):
                content = m.group(1)
                # 中文 / 英文都视为硬编码（短英文常量例外可加白名单）
                if re.search(r'[\u4e00-\u9fff]|[a-zA-Z]{4,}', content):
                    violations.append((f, content))
    return violations

violations = scan("src/ui")
if violations:
    print("❌ Hardcoded UI strings detected. Use tr() instead:")
    for f, s in violations: print(f"  {f}: {s!r}")
    sys.exit(1)
```

### 7.4 CI Job

```yaml
i18n_check:
  stage: lint
  script:
    - python3 tools/i18n_check.py
    - python3 tools/i18n_no_hardcode.py
  rules:
    - if: $CI_PIPELINE_SOURCE == "merge_request_event"
```

→ 任何硬编码字符串 / 缺 key / 占位符不一致 = MR 不能合。

## 八、动态文本与变量

### 8.1 数值替换

```gdscript
# 卡牌描述："造成 {damage} 点伤害"
var desc := tr("card.zhan.desc").format({"damage": 6})
# 输出："造成 6 点伤害"
```

### 8.2 多个变量

```gdscript
var msg := tr("battle.event.charge_burst").format({
    "charges": 5,
    "damage": 40,
})
# zh: "消耗 5 充能造成 40 伤害"
# en: "Consumed 5 charges to deal 40 damage"
```

### 8.3 复数处理

英文复数较麻烦。Godot 4.x 支持 `tr_n()` 函数：

```gdscript
# CSV
ui.card_count.one,"剩余 {count} 张牌","{count} card remaining"
ui.card_count.other,"剩余 {count} 张牌","{count} cards remaining"

# 代码
var n := deck.size()
label.text = tr_n("ui.card_count", "ui.card_count.other", n).format({"count": n})
```

中文不区分单复数，所以 zh_CN 列两条相同。

### 8.4 富文本（BBCode）

战斗 log 等需要变色 / 加粗：

```
combat.log.damage,"[color=red]{damage}[/color] 点伤害","[color=red]{damage}[/color] damage"
```

UI 用 `RichTextLabel.bbcode_text` 渲染，注意各语言 BBCode 标签必须保持一致（CI 校验）。

## 九、卡牌 / 事件文本规范

### 9.1 卡牌名翻译三原则

1. **直译**优先（"斩" → "Sword Strike"）
2. 如直译不传神，**意译**（"调息" → "Adjusting Breath"，不译为 "Rest"）
3. 五行属性 / 修仙概念走 Glossary

### 9.2 卡牌描述模板

中文：
```
造成 {damage} 点伤害；获得 1 充能。
```

英文：
```
Deal {damage} damage. Gain 1 Charge.
```

**规则**：
- 数值用 `{damage}` 而不是写死
- 升级版用相同 key 后缀 `_plus`
- 句末保持一致性（中文 "。" / 英文 "."）

### 9.3 事件文本

事件文本最长，建议拆细：

```csv
event.altar.title,"祭坛","The Altar"
event.altar.body,"你来到一个古老的祭坛...","You arrive at an ancient altar..."
event.altar.option_a,"献祭基础底座的第 1 槽位","Sacrifice slot #1 of base array"
event.altar.option_a_outcome,"获得遗物：修整时回血 50%","Gain Relic: Heal 50% during Recovery"
event.altar.option_b,"...","..."
```

每个选项 + 结果都是独立 key，便于复用。

## 十一、风险与对策

| 风险 | 严重度 | 对策 |
|------|------|------|
| 团队偷懒硬编码字符串 | 高 | CI 守卫强制阻塞 |
| AI 翻译错乱修仙术语 | 高 | Glossary 强制查表 + 人工校审 |
| CSV 大文件 Git diff 难看 | 中 | 按 scope 分多个 CSV 文件（如 `_battle.csv` / `_card.csv`） |
| 字体子集化漏字 | 中 | 字符集生成时扫描所有 zh_CN 文本，缺字自动加入 |
| 占位符在不同语言被翻译者误删 | 中 | CI 占位符一致性检测 |
| 后期加日韩需要重做布局 | 低 | UI Layout 用 `Container` 自动布局，不用绝对像素 |

## 十二、扩展到第三种语言（1.0 阶段参考）

加新语言的 SOP：

1. Sheet 加新列（如 `ja`）
2. AI 起草全部 `ja` 列
3. 人工校审 + Glossary 同步
4. 导出 `ja.csv`
5. `project.godot` 注册新 translation
6. 字体方案补日文字体（Noto Sans CJK JP）
7. UI 抽样测试（标题超长 / 字符宽度不同）
8. 上线

预计每新增一种语言：**3-5 周**（含校审）。
