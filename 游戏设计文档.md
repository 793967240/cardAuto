## 一、游戏一句话

> 玩家在爬塔过程中收集卡牌、底座、词条，构筑一条**自动循环执行的时间轴卡牌链**，与同样使用链条作战的敌人对抗，通过**协同发现**与**数值滚雪球**碾压关卡。

## 二、核心循环（Core Loop）

### 2.1 宏观循环（一局 Run）

```
进入地图 → 选择节点（战斗/精英/商店/事件/Boss）
          ↓
       触发节点内容
          ↓
       获得奖励（卡牌/底座/词条/遗物/金币）
          ↓
       构筑界面（拖拽卡牌进入底座、调整顺序、套用词条）
          ↓
       进入下一节点 …… → Boss → 通关 / 下一层
                                      ↓
                                     胜利 / 死亡（重开）
```

### 2.2 微观循环（一场战斗）

```
战斗开始
  ↓
玩家链条 & 敌人链条同时启动
  ↓
逐 tick 推进，依次执行链条上的卡牌
  ↓
所有卡牌打完 → 进入修整阶段（玩家不能行动，敌人继续输出）
  ↓
修整结束 → 链条从第一张重新开始
  ↓
循环，直到一方血量归零
```

### 2.3 构筑循环（玩家心流）

```
观察当前卡牌仓库与协同 → 看到新奖励 → 思考能否融入现有 build
                                          ↓
                                     选择性接受 / 拒绝
                                          ↓
                                     调整链条顺序与底座组合
                                          ↓
                                     在下一场战斗中验证效果
                                          ↓
                                     发现新协同 / 推翻原 build
```

## 三、核心机制

### 3.1 时间轴卡牌链条

**基础规则**：

- 玩家拥有若干"卡牌底座"，构成可放置卡牌的槽位
- 每张卡牌有 `cost` 属性，单位为 `tick`（基础值 1 tick = 0.5 秒）
- 战斗中，链条**从第一张牌开始顺序执行**，每张牌占用 `cost` 个 tick
- 全部卡牌执行完 → 进入**修整阶段** → 修整结束后**回到第一张**继续循环

**示例**：

```
玩家链条（1×6 底座）：
[攻击 1t][充能 2t][攻击 1t][强力一击 3t][防御 1t][回响 2t] | 修整 2t |
                                                        ↑
                                              此处循环回第一张
总循环时长：1+2+1+3+1+2+2 = 12 tick = 6 秒
```

### 3.2 修整阶段（Recovery Window）

- **核心定位**：弱点窗口
- 玩家链条全部执行完后强制进入
- 修整期间：玩家**完全不能行动**，敌人继续输出
- 修整时长：基础 2 tick，**可被卡牌效果或底座词条修改**，硬下限为 1 tick（不可归零）
- 设计意义：限制无脑堆长链条；让"短链快循环"和"长链强爆发"形成流派分化
- 部分卡牌可在修整阶段触发被动效果（如"修整时回血""修整时积累充能"）

#### 短链 vs 长链：核心流派张力

**短链不是"还没构筑完"，而是一种主动选择**：

- 一局 Run 中，玩家拿到的**强力卡（稀有/史诗/传说）数量有限**
- 链条越短 → 强力卡在单位时间内**触发频率越高** → DPS 上限越高
- 链条越长 → 摊薄了强力卡的占比，但提供更长的"协同 / 充能 / 延迟引爆"窗口

**两种流派的取舍**：

| 维度 | 短链流派 | 长链流派 |
|------|---------|---------|
| 强力卡复用频率 | **高**（每个循环都触发） | 低（被普通卡稀释） |
| 修整窗口暴露次数 | 多（频繁修整） | 少 |
| 协同/充能窗口 | 短，难铺垫 | 长，铺垫充足 |
| 失误容错 | 低（每张卡都关键） | 高（容得下过渡卡） |
| 适配协同类型 | 节奏型 / 加速型 / 回响型 | 充能型 / 延迟型 / 修整触发型 |

**设计推论**：

- 玩家**不必把所有底座槽位都填满**——主动留白（少接底座、少摆牌）是合法甚至高级的策略
- 这也是构筑界面**默认不强求所有底座连入链条**的设计动机（详见 §3.3 底座连接系统）
- 体修角色（极速短链 3-5 张）就是这条设计哲学的极端化体现

### 3.3 卡牌底座（Slot Base）

#### 底座类型

| 类型 | 槽位数 | 词条机制 | 获取方式 |
|------|------|---------|---------|
| **基础底座** | 1×6 | **每个槽位独立词条** | 开局自带，唯一 |
| **扩展底座** | 1×N (N=1~3) | **整个底座共享词条** | 战斗/事件/商店奖励 |

#### 底座规则

- 玩家可摆放的卡牌总数 = 所有底座槽位数之和
- 多个底座**首尾相连**形成完整链条（顺序由玩家决定）
- 基础底座的 6 个槽位**各有独立词条**，可单独强化（如"槽位3的牌 cost-1"）
- 扩展底座的整组共享 1 个词条（如"该组每张牌 +1 伤害"），强化时一起生效

#### 底座连接系统（珠子连线）

**核心机制**：

- **基础底座右侧**有 1 颗珠子（出口）
- **扩展底座左右两侧**各有 1 颗珠子（入口 / 出口）
- 玩家在构筑界面的**底座工作台**（左下区域）通过**拖拽珠子画线**，把扩展底座**逐节挂接**到基础底座的链条上
- **战斗中只有"被基础底座经珠子连通"的扩展底座才会进入时间链**；未连通的底座**整组不参与战斗**（包含里面的牌、共享词条都不生效）

**连接规则**：

- 链条形态为**单链**：基础底座 → 扩展底座 A → 扩展底座 B → ...（不允许分叉、不允许成环）
- 连线方向 = 链条执行方向（基础底座的牌先打，连下去的依次执行）
- **基础底座位置固定**，不可拖动；**扩展底座可在工作台内自由拖拽位置**，方便玩家走最短连线、避免连线交叉
- 玩家可随时**剪断**已有连线，重新组合

**设计意图**：

- 给"短链流派"一个**显式的策略表达**：玩家可以**故意只连接一部分扩展底座**，让强力卡轮转更快（详见 §3.2 短链 vs 长链）
- 把"扩展底座顺序"这个隐藏决策变成**可视化、可调试**的玩法动作
- 配合道韵·水墨连线的国风美术语言，**构筑动作本身就是视觉资产**
- 让"囤底座 → 等更好的底座 → 替换接入"成为合法的战术节奏

**新手保护**：

- 战斗开始前若存在"摆放但未连接"的底座，弹出**确认提示**："你有 N 个底座未接入链条，这些底座中的卡牌不会出战，确定开战？"
- 未连接的底座在工作台中**视觉灰显**，与已连接底座清晰区分

#### 底座裁切

- 通过特定事件/节点可"**裁切**"底座，**永久减少 1 个槽位**
- 通常作为**取舍机制**出现：失去槽位 ↔ 获得强力遗物/词条
- 基础底座可被裁切（1×6 → 1×5），扩展底座也可被裁切

**与底座连接系统的区别（互补关系，不冲突）**：

| 维度 | 底座连接（珠子连线） | 底座裁切 |
|------|---------------------|---------|
| 性质 | **可逆**的战术选择 | **不可逆**的永久改造 |
| 时机 | 构筑界面随时调整 | 仅事件/节点触发 |
| 代价 | 无（纯操作） | 通常需付出取舍（换遗物/词条） |
| 影响范围 | 底座**整组**生效 / 不生效 | 底座**单个槽位**永久消失 |
| 设计目的 | 让玩家**主动**调节链长 | 强制玩家面对**取舍** |

**典型联动场景**：

- 玩家在 Act 1 拿到一个 1×3 扩展底座但词条不强 → 暂时不连入链条（连接系统）
- Act 2 拿到更优的 1×2 底座 → 把 1×3 断开、接入 1×2（连接系统）
- Act 3 遇到事件"献祭基础底座槽位 6 → 获得强力遗物" → 永久裁切（裁切系统）

### 3.4 玩家不可被打断（设计哲学）

**核心规则**：玩家的链条一旦开始执行，**不会被任何敌人能力中断**。

**设计含义**：

- 游戏张力来源**不**是"玩家被打乱"的恐惧
- 张力来源是：
  - 时间不够（链条循环不完，玩家先死）
  - 修整窗口被敌人白嫖输出
  - 没有足够手段反制 boss 大招
- 整体调性是 **"解谜 + 构筑"**，而非 **"博弈 + 反应"**

**带来的玩家心智模型**：

- 构筑回报是**确定的**："我排好的链条一定会执行"
- 死亡归因清晰：是构筑问题，不是反应问题
- 完美贴合"轻度策略 + 背包乱炖"定位

**配套机制（必须实现）**：

- **战斗加速（MVP 必须）**：1x / 2x / 4x 倍速观看
  - **优先级最高**，在 MVP 阶段（玩法验证期）就必须加入
  - 自动战斗类游戏没有快进 = 无法验证玩法爽不爽
  - 默认建议 2x，玩家可手动切换
- **投降按钮**：明显劣势时直接判负，避免煎熬观看
- **快速重开**：死亡到下一局间隔 < 5 秒

> 注：这条规则只针对"打断当前卡的执行"。敌人仍可施加 debuff 影响**未来**的卡牌（如"易伤"使后续卡 cost+1），但**正在执行的卡不会停下**。

### 3.5 卡牌系统

#### 卡牌基本属性

```yaml
卡牌:
  名称: 例 "充能打击"
  cost: 2 tick
  类型: 攻击 / 防御 / 强化 / 控制 / 召唤 / 特殊
  标签: [火, 充能, 蹄发即...]    # 用于协同识别
  效果: 造成 X 伤害；获得 1 充能
  稀有度: 普通 / 稀有 / 史诗 / 传说
  消耗: 是否一次性
```

#### 协同机制类型（核心 Build 维度）

| 类型 | 示例 | Build 方向 |
|------|------|----------|
| **充能型** | 蓄力卡积累充能 → 爆发卡消耗充能造成 X×充能 伤害 | 前段铺垫，最后一张爆发 |
| **节奏型** | 攻击牌触发 N 次后获得 buff | 高密度短链 |
| **延迟型** | 种下"种子"，N tick 后引爆 | 与敌人大招读条对齐 |
| **加速型** | 催化剂使后续 N 张牌 cost -1 | 极速链条流派 |
| **回响型** | 复制前一张牌的效果 | 双倍爆发 |
| **共鸣型** | 链条中每有一张同标签卡，所有同标签卡 +1 伤害 | 单系堆叠 |
| **修整触发型** | 进入修整时全卡牌仓库获得 buff，下一轮爆发 | 长循环爆发流 |
| **打断型** | 蹄发即清除目标当前卡的进度 | 反制 boss 大招 |

#### 打断系统（详细规则）

- **触发**：使用打断标签的卡牌，蹄发即生效（无额外条件）
- **效果**：目标敌人**当前正在执行的卡进度归零**，但该卡仍留在链条上，从 0 重新蓄
- **限制**：
  - 被打断后，敌人获得 `打断免疫 N tick` buff（N 通常为 3-5），期间再次被打断牌命中无效（伤害正常结算）
  - **Boss 大招**可带 `打断抗性` 标签，仅"穿透打断"卡可生效
- **打断牌差异化维度**：
  - cost（轻打断 1t / 重打断 4t）
  - 是否带伤害
  - 是否消耗（一次性/可循环）
  - 是否 AOE
  - 附加效果（虚弱/标记/debuff延长）
  - 是否能穿透 `打断抗性`

### 3.6 敌人系统

#### 设计哲学

- **敌我同套规则**：敌人与玩家使用相同的"底座+卡牌"系统
- **完全公开信息**：玩家可查看敌人完整链条（默认每个敌人头顶展示 mini 链条；选中敌人时弹出**玩家+目标敌人**的横向甘特图对比）
- **固定模式**：普通敌人无随机抖动、无分支链条（轻度策略路线，玩家精确预判）
- **难度梯度**：

| 等级 | 链条长度 | 协同复杂度 | 特殊规则 |
|------|--------|----------|--------|
| 杂兵 | 2-3 张 | 单一标签 | 无 |
| 普通 | 4-5 张 | 简单协同 | 无 |
| 精英 | 5-7 张 | 多标签协同 | 部分大招带打断抗性 |
| Boss | 8-12 张 | 复杂多段协同 | 多张大招带打断抗性；可能带专属机制 |

#### 1 vs N 设定

- 战斗为玩家 1 名 vs 敌人 N 名（N=1~3）
- 普通战斗 N=1~2，精英 N=2~3，Boss 通常 N=1（聚焦单 boss 链条）
- N 个敌人**各自独立的链条与时间轴**
- 不设计跨敌人的"标记+引爆"协同（避免认知过载）

### 3.6 状态/标签系统

战斗中可施加的常见状态（待详细设计）：

- **虚弱**：受到伤害 +50%
- **易伤**：cost +1 tick
- **缓速**：链条推进速度 -50%
- **燃烧**：每 tick 受到 X 伤害
- **冻结**：链条暂停 N tick
- **标记**：受到下一次伤害 +X
- **充能**（玩家专属资源）：可被爆发卡消耗

## 四、爬塔结构

### 4.1 地图节点

参考杀戮尖塔的非线性地图，节点类型：

| 节点 | 内容 | 奖励 |
|------|------|------|
| 普通战斗 ⚔️ | 杂兵 1-2 个 | 卡牌（三选一） + 金币 |
| 精英战斗 💀 | 精英 1-2 个 | 卡牌 + 词条 / 遗物 + 金币 |
| 商店 🏪 | 购买卡牌/底座/词条/遗物/裁切服务 | - |
| 事件 ❓ | 文字事件 + 取舍选项 | 多样化（含底座、裁切机会） |
| 篝火 🔥 | 休息回血 / 强化卡牌 / 强化词条 | - |
| 神秘 ❔ | 随机以上类型 | 随机 |
| Boss 👑 | 单 boss，复杂链条 | 高品质遗物 + 进入下层 |

### 4.2 层级结构（初步）

- 共 3 层（Act 1/2/3）
- 每层 12-15 节点 + 1 Boss
- 每层主题、敌人池、卡牌池、底座池差异化
- 通关 Act 3 = Run 胜利

### 4.3 元进度（Meta Progression）

- 解锁新角色 / 新初始底座类型 / 新卡牌池 / 新事件
- 死了重开，但永久解锁内容保留
- 可选系统，不强制（避免新人门槛）

## 五、奖励池设计

### 5.1 奖励类型分布

战斗后奖励默认从以下池中三选一：

| 类型 | 出现概率（普通战斗） | 出现概率（精英） |
|------|----------------|----------------|
| 卡牌 | 60% | 40% |
| 底座 | 15% | 25% |
| 词条 | 15% | 20% |
| 遗物 | 5% | 15% |
| 金币 | 5% | - |

### 5.2 商店

商店内容固定包含：

- 3 张卡牌（不同稀有度）
- 1 个底座（随机大小）
- 2 个词条
- 1 个遗物
- "裁切服务"（消耗金币，永久移除链条上的某张牌）

### 5.3 事件取舍示例

```
【祭坛事件】
你来到一个古老的祭坛。它要求献祭，作为交换给予力量。

A. 献祭基础底座的第 1 槽位（裁切）
   → 获得遗物"修整时回血 50%"

B. 献祭基础底座的第 6 槽位（裁切）
   → 获得遗物"链条第一张牌 cost-1"

C. 献祭 30 金币
   → 获得稀有词条 1 个

D. 离开
```

## 六、UI/UX 关键设计

### 6.0 平台与设备约束

> **重要变更（v0.3）**：EA 版本走 Steam，**PC 为首发主平台**。安卓打包仅作开发期自测保留，不进商店。原"触屏优先"调整为"PC 键鼠优先 + 触屏作为兼容支持"。

**首发平台优先级**：

1. **Windows**（主要开发与测试目标，60% 测试时间）
2. **Linux / macOS**（Steam Deck 友好，10-15% 测试时间）
3. **Android**（仅 APK 自测，不上架商店）

**横屏布局（不变）**：

- 所有界面采用横屏布局，PC 下默认 16:9 窗口
- 时间轴是核心视觉，必须使用屏幕的横向空间
- 安卓自测包强制横屏；PC 不锁定窗口比例但以 16:9 为基准设计

**键鼠优先设计原则（首发主交互）**：

- **鼠标左键拖拽** 是核心交互方式（卡牌入槽、调整顺序）
- **鼠标悬停** 直接弹卡牌详情（无延迟，最佳体感）
- **右键** = 取消 / 返回 / 取出卡牌
- **滚轮** = 时间轴卷轴拖动 / 列表滚动
- **键盘快捷键**（必须支持）：
  - `空格` = 暂停 / 继续战斗
  - `1` `2` `3` = 切换战斗加速 1x / 2x / 4x

  - `Esc` = 设置菜单 / 返回上一界面
  - `Enter` = 确认默认按钮
- **最小可点击区域**：PC 下 32×32 px 即可

**触屏兼容（次要，仅供安卓自测包）**：

- 拖拽核心交互保持一致
- 鼠标悬停 fallback 为**长按 0.4 秒**弹详情
- 最小可点击区域 **48×48 dp**

**布局说明**：

- **下半区（底座工作台）**：玩家收集到的所有扩展底座放在这里，可**自由拖拽位置**（基础底座除外）
- **珠子 + 水墨连线**：基础底座右侧珠子作为链条**唯一出口**，玩家拖珠子画连线，将扩展底座**逐节挂接**进链条（详见 §3.3 底座连接系统）

- **右键卡牌** = 快速取出 / 放回侧栏（PC 加速操作）
- **拖拽珠子画连线** = 把扩展底座接入链条；**点击连线** = 剪断
- **拖拽扩展底座本体** = 调整在工作台中的物理位置（避免连线交叉，不影响连接关系）
- **顶栏"已接入链条 Xt"** = 只统计已连入链条的底座总时长（未连接的不计）
- **鼠标悬停卡牌** → 立即浮出详情卡片
- **鼠标悬停底座槽位** → 显示该槽位词条详情
- **鼠标悬停珠子** → 显示该底座是否生效、连接关系示意
- **长按 0.4s** 作为触屏 fallback
- **协同高亮**：链条中存在协同关系时，相关卡牌之间出现水墨连线（与连接连线视觉区分）
- **模拟战斗按钮**：进入"训练场"快速预览当前链条 vs 训练假人输出（含加速）
- **开战前校验**：若有未连接底座，弹出确认提示（详见 §3.3）
- **键盘快捷键**：`Esc` 返回 / `Enter` 进入下一节点

### 6.3 战斗加速（MVP 必须）

- 顶栏常驻**加速按钮**：1x / 2x / 4x 循环切换
- 默认 2x（自动战斗游戏的体感最优值）
- 加速影响**所有时间相关逻辑**（卡执行、动画、特效）
- 加速期间**关键事件可选自动暂停**（如玩家受到致命伤害预警），玩家可在设置中开关
- 战斗结算/动画快进时，**伤害数字飞出更密集**，避免信息丢失

## 七、角色体系（多角色，杀戮尖塔模式）

### 7.0 设计原则

- 参考杀戮尖塔：**多角色 + 流派差异化**
- 每个角色拥有：
  - **70% 专属卡池 + 30% 共享卡池**
  - **专属机制资源**（如剑修的"剑意"）
  - **不同初始卡牌仓库**
  - **不同初始遗物**（后续遗物系统加入后）
- 每个角色围绕**时间轴的不同利用方式**做差异化

### 7.1 首发角色规划（4 个）

#### 1. 剑修（Sword Cultivator）—— 入门角色

- **核心流派**：节奏型 + 共鸣型
- **专属机制**：**剑意**
  - 连续触发"剑"标签卡积累剑意
  - "斩"类卡消耗剑意爆发
  - 剑意上限 10
- **链条特征**：中等长度（5-8 张），剑标签密集
- **上手难度**：低
- **设计定位**：教学角色，让玩家理解时间轴+协同基础

#### 2. 符箓师（Talisman Master）—— 控制角色

- **核心流派**：延迟型 + 打断型
- **专属机制**：**符箓种子**
  - 贴出"符箓"种子，N tick 后引爆
  - 拥有大量"穿透打断抗性"的封印类卡
  - 是唯一能稳定打断 boss 大招的角色
- **链条特征**：短链 + 高密度打断
- **上手难度**：中
- **设计定位**：时间对齐玩法的最佳载体，反 boss 流

#### 3. 炼丹师（Alchemist）—— 堆叠爆发角色

- **核心流派**：充能型 + 修整触发型
- **专属机制**：**丹气**
  - 类似充能，但**修整阶段持续累积**
  - 爆发卡消耗丹气造成爆炸伤害
  - 唯一**主动延长修整反而获利**的角色
- **链条特征**：长链 + 长修整
- **上手难度**：中
- **设计定位**：颠倒"修整=弱点"的反直觉流派

#### 4. 体修（Body Cultivator）—— 极速爆发角色

- **核心流派**：加速型 + 自损强化
- **专属机制**：**淬体**
  - 消耗自身血量获得 buff
  - 大量"催化剂"系卡牌，链条 cost 极低
  - 链条最短但循环最快
- **链条特征**：极速短链（3-5 张），每秒触发数倍于其他角色
- **上手难度**：高
- **设计定位**：高风险高回报，资深玩家挑战

### 7.2 角色解锁节奏

- **剑修**：初始解锁
- **符箓师**：剑修通关 Act 1 后解锁
- **炼丹师**：剑修通关 Act 3 后解锁
- **体修**：任意角色通关高难度后解锁

### 7.3 共享卡池设计

- 占总卡池约 30%
- 包含基础攻防、状态施加、通用功能性卡牌
- 不包含任何角色专属机制（剑意/符箓/丹气/淬体不互通）

## 九、卡牌升级机制（篝火强化）

### 9.1 基础规则

- 参考杀戮尖塔：**每张卡牌设计 1 个升级版本**（"+"形态）
- 每张卡只可升级一次（升级后无法再升）
- 升级途径：
  - 篝火节点选择"修炼"
  - 部分事件可触发免费升级
  - 部分遗物提供升级机会
- 升级是**永久的**（直到 Run 结束）

### 9.2 升级方向（结合时间轴系统）

不同卡牌应有不同的升级方向，避免单一性：

#### A. **数值强化型**（最常见）

```
斩          cost 1t, 6 伤
斩+         cost 1t, 9 伤
```

#### B. **cost 优化型**（最强力，需谨慎使用）

```
火球        cost 2t, 8 伤
火球+       cost 1t, 8 伤    ← 节省 1 tick
```

#### C. **效果扩展型**

```
催化剂      cost 1t, 后续 3 张牌 cost-1
催化剂+     cost 1t, 后续 4 张牌 cost-1
```

#### D. **协同放大型**

```
共鸣·火     链条中每张火卡 +1 伤
共鸣·火+    链条中每张火卡 +2 伤
```

#### E. **附加效果型**

```
封印        cost 2t, 打断目标当前卡
封印+       cost 2t, 打断目标当前卡 + 施加 2 层"道伤"
```

#### F. **修整影响型**（时间轴特色）

```
余烬        cost 1t, 链尾卡，修整时再次触发本卡
余烬+       cost 1t, 链尾卡，修整时再次触发本卡 + 修整时长 -1
```

### 9.3 升级设计原则

- **强度提升约 30-50%**，不能颠覆原牌定位
- 升级**不应改变卡牌的核心功能**（攻击牌不会变成防御牌）
- **传说级卡牌**升级后效果更夸张，作为后期爽点
- **打断牌**升级方向倾向 E（附加效果），而非 B（cost 优化），避免无限锁招

### 9.4 篝火节点的可选项

```
【篝火 / 静修之地】

A. 调息（恢复 30% 最大生命）
B. 修炼（强化一张卡牌）
C. [遗物可解锁] 顿悟（强化两张牌但损失 5 HP）
```

## 十、核心爽点设计

### 7.1 协同发现

游戏需要在卡牌池中预埋大量"看似无关、组合起来爆炸"的协同：

**示例 Build：充能爆发流**

```
[蓄力 1t, +1充能]
[蓄力 1t, +1充能]
[催化剂 1t, 后续3张牌cost-1]
[蓄力 0t（被催化）, +1充能]
[蓄力 0t（被催化）, +1充能]
[爆发 0t（被催化）, 消耗所有充能, 每点充能造成 8 伤害]
→ 5 充能 × 8 = 40 伤害爆发
```

**示例 Build：共鸣火系流**

```
链条中有 5 张"火"标签卡 → 每张火卡 +5 伤害
[火球 1t, 6+5=11伤]
[火舌 1t, 4+5=9伤]
[燃烧 1t, 施加3层燃烧]
[爆裂 2t, 引爆所有燃烧, 每层造成5伤]
[余烬 1t, 修整时再次触发本链条最后一张牌]
```

### 7.2 数值滚雪球

后期 Run 应该让玩家有"我无敌了"的瞬间：

- 通过堆叠词条，单张牌伤害从 5 → 50
- 通过链条循环加速，每秒触发次数从 1 → 5
- 通过修整缩短到极限，几乎无停顿循环输出

### 7.3 取舍张力

底座裁切、事件抉择、卡牌移除等机制持续考验玩家：

- "我要不要为了一个强词条砍掉一个槽位？"
- "这张神卡 cost 太高，会拖慢我的循环节奏"
- "买这个遗物还是攒钱进 boss 战前？"

---

## 十一、平衡性设计原则

### 8.1 数值框架（初稿）

| 项目 | 数值 |
|------|------|
| 1 tick | 0.5 秒 |
| 普攻类卡 cost | 1-2 tick |
| 强力技能 cost | 3-4 tick |
| 蓄力/瞬发 buff cost | 0-1 tick |
| 一轮链条总时长 | 8-15 tick (4-7.5秒) |
| 修整基础时长 | 2 tick (1秒) |
| 修整硬下限 | 1 tick |
| 普通敌人血量（Act 1） | 30-50 |
| 玩家初始血量 | 80 |
| 战斗目标时长 | 30-60 秒 (5-8 轮链条) |
| 单局 Run 时长目标 | 45-75 分钟 |

### 8.2 强度天花板

- **修整时长**：可缩短但不可消除（硬下限 1 tick）
- **链条总时长**：无上限（玩家可堆出超长链条）
- **单卡伤害**：可堆叠（无硬上限，但需要多重词条+协同）
- **打断频率**：被打断免疫 N tick 限制无限锁招
- **充能上限**：可设上限（如 99）防止数值溢出

### 8.3 反"统治性策略"机制

- 卡牌"消耗"标签：强力一次性卡用完即消失
- 词条互斥：某些词条不能同时挂在同一槽位
- 底座槽位限制：玩家不能无限扩展（一局 Run 最多累计 X 个槽位）

## 十二、MVP（最小可玩原型）开发路径

### 阶段 1：核心战斗循环验证（最优先）

**保留**：

- 1×6 基础底座（无词条）
- 10-15 张基础卡牌（覆盖 3 种协同类型）
- **剑修** 1 个角色，固定起手卡牌仓库
- 5 种敌人（1 杂兵 + 2 普通 + 1 精英 + 1 boss）
- 战斗界面（含时间轴 UI、横屏布局）
- 构筑界面（拖拽 + 实时模拟）
- **战斗加速 1x/2x/4x（必须）**
- 投降按钮
- 一条线性"爬塔"路径（10 个节点）
- 篝火节点（仅"调息回血" + "强化卡牌"两个基础选项）

**砍掉（暂不实现）**：

- 底座词条系统
- 扩展底座
- 裁切机制
- 事件文本（事件节点可保留，但只显示纯选项不写文案）
- 商店
- 遗物系统
- 元进度
- 其他 3 个角色

**目标**：验证"时间轴卡牌+自动战斗"的核心循环爽不爽。

### 阶段 2：构筑深度扩展

加入：

- 扩展底座（1×1, 1×2, 1×3）
- 底座词条（独立词条池 + 共享词条池）
- 卡牌库扩展到 50-80 张
- 协同机制扩展到 8 种
- 篝火节点完整化

### 阶段 3：Roguelike 外壳

加入：

- 完整爬塔地图（非线性）
- 事件节点 + 商店
- 事件文本与剧情包装
- 裁切机制
- 遗物系统（含影响底座/战斗节奏的遗物）
- 3 层结构

### 阶段 4：多角色与 Meta

- **符箓师 / 炼丹师 / 体修** 3 个角色逐步解锁
- 元进度系统
- 平衡性调整与 boss 设计
- 视觉效果与音效深度打磨

### 阶段 5：平台扩展

- PC 端（键鼠适配）
- 手柄支持
- 性能优化与多分辨率适配

## 十三、设计决策记录（与原始构思的差异）

| 项目 | 原始构思 | 当前决定 | 原因 |
|------|--------|--------|------|
| 战斗博弈深度 | 未定 | 轻度策略向 | 受众更广，符合背包乱炖路线 |
| 敌人时间轴 | 未定 | 完全公开 + 选中详情 | 玩家完美信息预判 |
| 敌人 AI | 未定 | 固定模式（无抖动/分支） | 简化认知负载 |
| 修整时长机制 | 未定 | 可被卡/词条修改的基础值 | 给构筑足够维度 |
| 打断后果 | 未定 | 进度归零，留在当前牌 | 简洁、可预测、不反智 |
| 打断防连锁 | 未定 | 打断免疫 N tick + boss 大招打断抗性 | 平衡且有梯度 |
| 打断触发条件 | 未定 | 蹄发即打断（无额外条件） | 易于上手 |
| 玩家被打断 | 未定 | **完全不可被打断** | "解谜+构筑"调性，张力来自时间不够而非反应 |
| 1 vs N 中的 N | 未定 | N=1~3 | 避免认知过载 |
| 角色体系 | 未定 | **多角色（杀戮尖塔模式）**，4 个角色不同流派 | 内容扩展性 + 玩家选择 |
| 卡牌升级 | 未定 | **每卡一个 +版本（杀戮尖塔模式）** | 成熟方案，多种升级方向 |
| 美术风格 | 未定 | **水墨修仙** | 文化辨识度 + 与机制语言天然契合 |
| 音乐节奏同步时间轴 | 加分项 | 初版不强制同步 | 优先验证玩法，避免过度设计 |
| 平台 | 未定 | **触屏优先（手机），后续 PC + 手柄** | 战斗自动化适合移动端 |
| 屏幕方向 | 未定 | **强制横屏** | 时间轴需横向空间 |
| 战斗加速 | 未定 | **MVP 必须，1x/2x/4x** | 自动战斗类游戏标配，玩法验证刚需 |
| 自走棋多角色 | 似乎要 | 单玩家角色（"自走棋"仅指自动战斗） | 已确认 |

## 十五、风险评估

| 风险点 | 严重度 | 应对方向 |
|------|------|--------|
| 时间轴 UI 设计难度高 | 高 | 早期投入 UI 原型迭代 |

| 内容量不足导致重复感 | 中 | 卡牌/底座/事件量必须管够 |
| 系统过多（卡/底座/词条/遗物） | 中 | 词条与遗物职能要清晰区分 |
